Crée une application mobile React Native complète pour la plateforme de commande et livraison de gaz de cuisine à domicile, connectée au backend Django REST API créé précédemment.
Intégration avec le Backend Django :

Consommer toutes les API REST du backend Django
Authentification JWT (login, register, refresh token)
Endpoints à intégrer :

/api/auth/ - Authentification et gestion utilisateurs
/api/stations/ - Gestion des stations de service
/api/bouteilles/ - CRUD bouteilles de gaz (avec nom commercial obligatoire)
/api/commandes/ - Gestion des commandes
/api/paiements/ - Traitement des paiements
/api/livreurs/ - Gestion des livreurs
/api/zones/ - Zones de livraison
Tous les autres endpoints définis dans le backend Django



Architecture Technique :

React Native (dernière version stable)
Navigation : React Navigation 6+
Gestion d'état : Redux Toolkit ou Context API + useReducer
API : Axios avec intercepteurs configurés pour le backend Django
Base URL : variable d'environnement pointant vers le serveur Django
Authentification : JWT (stockage sécurisé avec AsyncStorage ou SecureStore)
Carte/Géolocalisation : react-native-maps + react-native-geolocation
Paiement : intégration SDK correspondant au backend (Stripe, Flutterwave, etc.)
Notifications : react-native-push-notification ou Firebase Cloud Messaging
UI : React Native Paper ou NativeBase (design moderne)

Interfaces par Type d'Utilisateur (selon les rôles du backend) :
1. AUTHENTIFICATION (Commune) :

Écran de bienvenue/splash
Connexion (email + mot de passe) → appel API /api/auth/login/
Inscription avec sélection de rôle (Client, Livreur, Station) → /api/auth/register/
Récupération de mot de passe → /api/auth/password-reset/
Validation du compte
